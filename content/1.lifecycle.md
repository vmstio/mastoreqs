::tip
For the purposes of this document, when applied to dependencies the term "End of Life" is standardized to mean when there are no further updates of any kind, and/or no support available for a released version of Mastodon or it's dependencies.
::

| Mastodon  | Status         |                                            |
| ------------- | ------------------ | :----------------------------------------: |
| 4.6.x         | Development Branch | :icon{name="i-fluent-emoji-flat-check-box-with-check"} |
| 4.5.x         | Supported Release  |  :icon{name="i-fluent-emoji-flat-check-box-with-check"}   |
| 4.4.x         | Supported Release  |  :icon{name="i-fluent-emoji-flat-check-box-with-check"}   |
| 4.3.x         | Pending EOL        |       :icon{name="i-fluent-emoji-flat-warning"}        |
| 4.2.x         | Pending EOL        |       :icon{name="i-fluent-emoji-flat-warning"}        |
| 4.1.x & prior | End of Life        |      :icon{name="i-fluent-emoji-flat-cross-mark"}      |

::collapsible
| Key                                        |                                            |
| ------------------------------------------ | ------------------------------------------ |
| :icon{name="i-fluent-emoji-flat-check-box-with-check"}    | Supported in this version                  |
| :icon{name="i-fluent-emoji-flat-warning"}              | End of life, use with caution |
| :icon{name="i-fluent-emoji-flat-cross-mark"}           | Not supported in this version              |
::

References: [Mastodon Security Policy](https://github.com/mastodon/mastodon/security)

::caution
End of Life versions of Mastodon do not receive any features, bug fixes, and generally no security updates except at the discretion of the core development team.
::

::warning
Any dependencies listed for development versions (nightly/alpha) of Mastodon are based on tracking the development process of the `main` branch on GitHub and the project Discord.
These and are subject to change at any time as project requirements change.
::
