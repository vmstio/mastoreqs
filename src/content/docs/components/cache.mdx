Mastodon requires administrators to choose a Redis-compatible backend for operation. It
supports Redis or Valkey, and one of these must be present and configured.

:::note
Other Redis API-compatible providers, such as [DragonflyDB](https://www.dragonflydb.io),
may also work with Mastodon. These alternatives are not tracked as part of this matrix, so
consult your chosen software's own documentation for its Mastodon compatibility.
:::

## Valkey

<SupportMatrix component="cache" matrix="valkey" />

References: [Valkey Releases](https://valkey.io/topics/releases/)

```bash
# Check running version
valkey-server --version
```

## Redis

<SupportMatrix component="cache" matrix="redis" />

References: [Redis End of Life](https://endoflife.date/redis), [Redis Security Policy](https://github.com/redis/redis/security)

```bash
# Check running version
redis-server --version
```

:::danger
Mastodon has not required upgrades of Redis for many years, but this has changed with
versions 4.4+. Please review the available versions in your deployment carefully before upgrading.
:::
